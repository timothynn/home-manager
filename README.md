# home-manager

Opinionated NixOS + Home Manager superflake for `tim@nixos`.

- **OS**: NixOS `nixos-unstable` (aiming at 26.05). `system.stateVersion = "25.05"`; Home Manager `home.stateVersion = "26.05"`.
- **WM**: Hyprland `0.54.3` (flake-pinned — not master).
- **Theme**: Catppuccin Mocha (accent=mauve), wired through `catppuccin-nix` to every program that has a module (kitty, bat, fzf, atuin, yazi, zellij, lazygit, zsh syntax highlighting, delta, GTK, Qt, cursors, hyprland, rofi, waybar, mako, SDDM, hyprlock).
- **Hosts**: `default` (auto), `laptop`, `desktop` — all three share `hosts/default/configuration.nix` and switch behaviour via `modules/system/host-role.nix`.
- **Secrets**: `sops-nix` + `age`. The flake imports `sops-nix` unconditionally; `modules/system/secrets.nix` only wires secrets up if `secrets/secrets.yaml` exists, so the flake still evaluates on a fresh clone.

---

## Layout

```
flake.nix                # inputs + nixosConfigurations + homeConfigurations + devShells
hosts/default/
    configuration.nix    # top-level system config, imports every modules/** file
    hardware-configuration.nix  # generated by nixos-generate-config — DO NOT hand-edit
modules/                 # NixOS-level modules (system.*/services.*/security.*)
    system/              # networking, fonts, locale, secrets, host role, base config
    services/            # seabury-vpn, swaylock
    desktop/             # sddm, hyprland enablement, hyprlock, mako
    programs/            # rofi, firefox, git, kitty, waybar, oh-my-posh, wofi
    theme/               # GTK, Qt, cursors
home/                    # Home Manager user-scope modules
    desktop/             # hyprland user config, hyprpaper, waybar
    terminal/            # kitty
    shell/               # zsh, nushell, starship, cli-tools, node, github-sync, dashboard
    editor/              # neovim, helix
    theme/               # catppuccin global defaults
home.nix                 # Home Manager entry point (imported by flake.nix)
wallpapers/              # catppuccin-mocha-ridges.jpg (default wallpaper)
```

## Prerequisites

1. **A working NixOS install** (from the graphical installer, an installer ISO, or any existing box — the flake does not partition or install the OS).
2. **Flakes enabled** on the machine you'll run `nixos-rebuild` from:
   ```bash
   sudo mkdir -p /etc/nix
   echo 'experimental-features = nix-command flakes' | sudo tee -a /etc/nix/nix.conf
   ```
   (The flake itself also sets this, so after the first `switch` the line above is redundant — it only matters for bootstrap.)
3. **git** and **nix** (both ship by default on a fresh NixOS install).

## First-time setup

```bash
# 1. Clone into the location nixos-rebuild expects a flake to live.
sudo mkdir -p /etc/nixos
sudo chown "$USER:users" /etc/nixos
git clone https://github.com/timothynn/home-manager.git /etc/nixos
cd /etc/nixos

# 2. Replace the checked-in hardware-configuration.nix with the one
#    generated for THIS machine. The repo ships a placeholder so the
#    flake evaluates; it is almost certainly wrong for your disk layout.
sudo nixos-generate-config --show-hardware-config \
  > hosts/default/hardware-configuration.nix

# 3. If you want Hyprland at a newer tag than v0.54.3, bump the ref in
#    flake.nix, then:
nix flake lock --update-input hyprland

# 4. Build & switch. Pick one of the three hosts:
sudo nixos-rebuild switch --flake .#default      # auto laptop/desktop via DMI chassis
# sudo nixos-rebuild switch --flake .#laptop
# sudo nixos-rebuild switch --flake .#desktop
```

Home Manager is wired in as a NixOS module (`useGlobalPkgs = true; useUserPackages = true;`), so the user profile is built as part of `nixos-rebuild switch` — you do **not** need to run `home-manager switch` separately for `tim`. There is also a standalone `homeConfigurations.tim` output if you want to target HM by itself:

```bash
nix run home-manager/master -- switch --flake .#tim
```

## Post-rebuild steps

### 1. Seabury VPN (one-time, per machine)

The `seabury-vpn.service` unit is guarded by `ConditionPathExists=/etc/openvpn/client/keys/client.conf` and stays a silent no-op until you drop the file in place. Neither `client.conf` nor the password is committed to this repo — both are sensitive and live outside git.

```bash
# (a) Grab client.conf from gitlab.seaburymro.com / Seabury IT
sudo install -m 0600 -o root -g root client.conf \
  /etc/openvpn/client/keys/client.conf

# (b) Wire in auth-user-pass and Seabury's MTU hint
sudo tee -a /etc/openvpn/client/keys/client.conf >/dev/null <<'EOF'
auth-user-pass /etc/openvpn/client/keys/auth.txt
tun-mtu 1380
EOF

# (c) Create the credentials file (username on line 1, password on line 2)
sudo install -m 0600 -o root -g root /dev/null /etc/openvpn/client/keys/auth.txt
sudoedit /etc/openvpn/client/keys/auth.txt

# (d) Enable + start. The unit takes over on every boot from here on.
sudo systemctl enable --now seabury-vpn.service
journalctl -u seabury-vpn -f
```

If you'd rather run the tunnel interactively, `openvpn3` is also installed:

```bash
openvpn3 session-start  --config /etc/openvpn/client/keys/client.conf
openvpn3 session-manage --config /etc/openvpn/client/keys/client.conf --disconnect
```

To manage the credential file declaratively instead of by hand, encrypt it with sops (`sops-nix` is already imported) and point `sops.secrets.seabury-auth.path = "/etc/openvpn/client/keys/auth.txt";` at `mode = "0400"; owner = "root";`.

### 2. npm globals without sudo

After the first rebuild **and a fresh shell** (so `home.sessionPath` picks up):

```bash
npm config get prefix          # /home/tim/.npm-global
npm install -g cowsay          # no sudo, no EACCES
which cowsay                   # /home/tim/.npm-global/bin/cowsay
```

### 3. .NET SDKs (currently disabled)

.NET is commented out. To re-enable, uncomment the three blocks marked `.NET is disabled for now` in:

- `home.nix` (let-binding + `home.packages` append)
- `flake.nix` (`permittedInsecurePackages` + `dotnetSdks` + `dotnet` devShell)
- `hosts/default/configuration.nix` (`permittedInsecurePackages`)

Once uncommented, `home.nix` installs `dotnetCorePackages.combinePackages [sdk_6_0 sdk_8_0 sdk_10_0]` so one `dotnet` binary dispatches across all three. The SDK list is `lib.optional`-guarded so missing versions don't break eval.

```bash
dotnet --list-sdks             # 6.0.x, 8.0.x, 10.0.x (minus anything missing)
```

**.NET 6 is EOL.** The pinned allowance is `dotnet-sdk-6.0.428`. If your nixpkgs channel has a different patch version, eval will fail with an "insecure" error naming the real version — either copy that version string into `flake.nix` and `hosts/default/configuration.nix`, or switch to a predicate:

```nix
nixpkgs.config.allowInsecurePredicate =
  p: pkgs.lib.hasPrefix "dotnet-sdk-6" (pkgs.lib.getName p);
```

### 4. Secrets (optional)

If you want the `sops-nix` wiring to do anything on this machine:

```bash
# Generate an age key (once per machine)
mkdir -p ~/.config/sops/age
age-keygen -o ~/.config/sops/age/keys.txt

# Then create secrets/secrets.yaml encrypted for that key.
# See https://github.com/Mic92/sops-nix for the full sops workflow.
```

`modules/system/secrets.nix` detects `secrets/secrets.yaml` via `builtins.pathExists` and silently skips the wiring if it's absent, so this step is entirely optional.

### 5. Hyprlock sanity check

Lock the session (`Super+L` by default) and type your password. It should unlock. If it does not, `journalctl --user -b | grep -i pam` usually gives the reason — `security.pam.services.hyprlock = {};` lives in `modules/desktop/hyprlock.nix` and is the piece that makes PAM consult the correct stack.

### 6. Rotate the password you may have pasted in chat

Devin sessions are persisted. If you pasted your Seabury password in chat while bringing this PR up, **rotate it** and put the new one into `/etc/openvpn/client/keys/auth.txt`.

## Day-to-day

```bash
# Rebuild after config edits
sudo nixos-rebuild switch --flake /etc/nixos#default

# Dry-run eval without activating
sudo nixos-rebuild dry-build --flake /etc/nixos#default

# Home Manager only (standalone output)
nix run home-manager/master -- switch --flake /etc/nixos#tim

# Update all inputs (or pick one with --update-input)
nix flake update
nix flake lock --update-input hyprland

# Garbage collect
sudo nix-collect-garbage -d       # also removes old generations
nix-collect-garbage -d             # user profile only

# Dev shells (one per ecosystem)
nix develop /etc/nixos#rust
nix develop /etc/nixos#python
nix develop /etc/nixos#node
# nix develop /etc/nixos#dotnet  # disabled; see § 3
```

The `auto-switch` flake app picks `laptop` vs `desktop` by reading `/sys/class/dmi/id/chassis_type`:

```bash
nix run /etc/nixos#auto-switch
```

## Troubleshooting

**Intel AC 3160 Wi-Fi stays `unavailable`.** After rebuild, reboot (modprobe options only take effect on a clean module reload), then:

```bash
rfkill list
dmesg | grep -iE 'iwlwifi|iwlmvm'
nmcli -f all device show wlp85s0
systemctl status iwd NetworkManager
```

The custom `iwlwifi` options (`11n_disable=8`, `bt_coex_active=0`) are in `modules/system/default.nix`.

**`catppuccin.<program>.enable does not exist`.** You're on a `catppuccin-nix` commit that doesn't expose that program's module yet. Comment the offending `catppuccin.<program>.enable = true;` line out and rebuild — the program still works, it just won't be themed by that module. (PR #3 moved every catppuccin opt from the `programs.<p>.catppuccin.*` form to the correct top-level `catppuccin.<p>.*` form.)

**`package dotnet-sdk-6.X.Y is marked insecure`.** You uncommented the .NET blocks and the pinned version in `permittedInsecurePackages` drifted. See the .NET section above.

**Hyprland fails to start after `flake update`.** This used to be the default story when `hyprland` tracked master. The flake now pins `?ref=v0.54.3`; bump the ref in `flake.nix` explicitly, run `nix flake lock --update-input hyprland`, and re-read the Hyprland 0.51/0.53+ breaking changes before jumping more than a minor version.

**`WLR_NO_HARDWARE_CURSORS`.** Don't re-add it unless you actually have an NVIDIA GPU. On Intel graphics it forces software cursors and introduces flicker. The deliberately-removed line has a comment in `modules/desktop/hyprland.nix` so it doesn't come back.

**Binary cache not helping — Hyprland is building from source.** Confirm `substituters` / `trusted-public-keys` in `nix.settings` include `hyprland.cachix.org` (they do, in `hosts/default/configuration.nix`), and that your user is in `trusted-users` or `@wheel`. First-time trust requires `sudo nixos-rebuild switch` (not `nix build`).

**Out-of-tree kernel modules.** `boot.kernelPackages = pkgs.linuxPackages_latest` can outrun third-party modules (NVIDIA driver, VirtualBox host modules, ZFS, DKMS). If `nixos-rebuild` fails at the module build step, pin back to `pkgs.linuxPackages` or a specific LTS in `modules/system/default.nix`.
